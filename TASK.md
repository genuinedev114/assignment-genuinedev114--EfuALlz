
# Invoice Processing System

Build a full-stack application for uploading and processing invoices. We want to see how you design under constraints - clear decisions, clean architecture, async state that works, and a UI that does the job. Beyond that, you're free to add any additional functionality or choose any approach you see fit.


## What we expect to see

- Invoice upload (PDF or image)
- A list of uploaded invoices with their current status
- Real-time status updates — no manual refresh
- Extracted invoice data visible once processing completes
- A retry mechanism for failed invoices
- An AI assistant the user can talk to — ask about invoice statuses, query results
- The AI assistant can take actions, not just answer questions

Each invoice should move through: `uploaded → processing → completed / failed`


## Deliverables

- A `README` covering your approach, what you'd improve with more time, and how you used AI tools during development
- Any AI-generated artifacts (prompts, specs, planning docs) — leave them in the repo as-is
- Clear instructions on how to run and test the project
